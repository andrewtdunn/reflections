What happens when you initialize a repository? Why do you need to do it?

  You register the root directory to be tracked.

How is the staging area different from the working directory and the repository? What value do you think it offers?

  It allows you to commit only parts of the working directory.

How can you use the staging area to make sure you have one commit per logical change?

  You can only stage files together that are part of the same change.

What are some situations when branches would be helpful in keep your history organised? How
would branches help?

  When you have multiple new features to help, you can work on each feature in a separate branch.

How do the diagrams help you visualize the branch structure?

  It tells you how the branches are related, and which commits are in which branch.
